## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | send_ops_alert error message | mitigate | D-02 specifies error TYPE only (`type(exc).__name__`), not full exception message. This prevents leaking file paths, credentials, or internal state in stack traces to Discord. The full error is logged server-side via `logger.error`. |
| T-09-02 | Denial of Service | error_count cap | mitigate | D-03/D-04 cap at 3 individual error posts + 1 overflow summary per scan run (max 4 Discord API calls for errors). Prevents a scan with many failing tickers from flooding the Discord channel or hitting Discord rate limits. |
| T-09-03 | Spoofing | error message content | accept | The `ticker` variable comes from the watchlist (operator-controlled file) and `type(exc).__name__` is a Python-internal string. Neither is user/attacker-controlled input. No injection risk into Discord messages. |
