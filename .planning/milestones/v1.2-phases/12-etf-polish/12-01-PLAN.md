```python
def _parse_scan_times() -> list[str]:
    """Parse SCAN_TIMES env var (comma-separated HH:MM). Falls back to SCAN_HOUR:SCAN_MINUTE."""
    raw = os.getenv("SCAN_TIMES", "").strip()
    if raw:
        return [t.strip() for t in raw.split(",") if t.strip()]
    hour = int(os.getenv("SCAN_HOUR", "9"))
    minute = int(os.getenv("SCAN_MINUTE", "0"))
    return [f"{hour:02d}:{minute:02d}"]
```

Current main.py:configure_scheduler (lines 40-49) — already job-fn-agnostic but hardcodes job id prefix `scan_`:

```python
def configure_scheduler(scheduler: BackgroundScheduler, config: Config, job_fn) -> None:
    """Register one scan job per time in config.scan_times."""
    for i, time_str in enumerate(config.scan_times):
        hour, minute = map(int, time_str.split(":"))
        scheduler.add_job(
            job_fn,
            trigger=CronTrigger(hour=hour, minute=minute),
            id=f"scan_{i}",
            replace_existing=True,
        )
```

Per D-03 this function must be reused for the ETF job. Because it hardcodes `f"scan_{i}"`, we must EITHER:
- (a) add an optional `job_id_prefix: str = "scan"` parameter and pass `"etf_scan"` from the ETF call, OR
- (b) add an optional `times: list[str] | None = None` param that overrides `config.scan_times`, PLUS the prefix.

This plan chooses (a)+(b): add `times` and `job_id_prefix` kwargs so callers supply `config.etf_scan_times` and `"etf_scan"`. Default args preserve backward compatibility for the existing stock scan call site.

Current main.py:on_ready (lines 525-536) — stock scheduler call to mirror:

|-----------|----------|-----------|-------------|-----------------|
