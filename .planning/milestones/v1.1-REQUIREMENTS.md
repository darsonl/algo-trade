# Requirements: Algo Trade Bot

**Defined:** 2026-04-06
**Core Value:** The bot must never place a real order without explicit human approval via Discord.

## v1.1 Requirements

Requirements for v1.1 milestone. Each maps to roadmap phases (7–8).

### ETF Scan Separation

- [x] **ETF-01**: User can classify watchlist.txt tickers into stocks and ETFs via `partition_watchlist()` using yfinance `quoteType`, with static allowlist fallback for rate-limit edge cases
- [x] **ETF-02**: User can trigger `/scan_etf` Discord slash command to run an ETF-specific scan that skips the fundamental filter entirely
- [x] **ETF-03**: ETF scan uses a `build_etf_prompt` analyst prompt focused on trend/RSI/MACD/expense ratio rather than P/E and earnings growth
- [x] **ETF-04**: ETF recommendations appear as Discord embeds with ETF-specific fields (price, RSI, MA50 trend, expense ratio) via `build_etf_recommendation_embed`
- [x] **ETF-05**: ETF recommendations are stored in the recommendations table with `asset_type='etf'` column (additive migration)
- [x] **ETF-06**: ETF universe is configured via `etf_watchlist.txt`; `_ETF_ALLOWLIST` in `fundamentals.py` removed after `partition_watchlist` is verified end-to-end

### Asyncio Event Loop Hardening

- [x] **ASYNC-01**: All blocking yfinance calls in `run_scan` buy pass (`fetch_fundamental_info`, `fetch_news_headlines`, `fetch_technical_data`) wrapped in `asyncio.to_thread`
- [x] **ASYNC-02**: All blocking yfinance calls in `run_scan` sell pass (`fetch_news_headlines`, `fetch_technical_data`) wrapped in `asyncio.to_thread`
- [x] **ASYNC-03**: `partition_watchlist` wrapped in `asyncio.to_thread` at its Phase 7 call site — not deferred to Phase 8
- [x] **ASYNC-04**: `get_top_sp500_by_fundamentals` hotfix formalized with `# already wrapped` comments; Phase 8 audit documents all already-wrapped sites before adding new wraps

## Future Requirements

### ETF Enhancements

- **ETF-07**: Scheduled ETF scan at a time offset (e.g. 9:05 AM) to avoid yfinance rate-limit contention with the stock scan
- **ETF-08**: Separate ops alert prefix `[ETF]` to distinguish ETF scan silence from stock scan silence
- **ETF-09**: Expense ratio threshold filter (flag ETFs with >0.50% expense ratio in embed)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Parallel ticker scanning via asyncio.gather | Explicitly out of scope per PROJECT.md |
| Dynamic ETF universe from external API | Static etf_watchlist.txt sufficient for single-operator use |
| ETF_SCAN_TIMES config field | Manual /scan_etf sufficient until scheduled ETF scan is validated |
| Multi-user approval for ETF recommendations | Single operator assumed |
