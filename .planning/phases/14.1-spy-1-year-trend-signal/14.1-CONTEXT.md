# Phase 14.1: SPY 1-year trend signal - Context

**Gathered:** 2026-04-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Add a 1-year SPY trend signal to `fetch_macro_context()` alongside the existing 1-month signal, rename `spy_trend` → `spy_trend_1m` throughout so both signals coexist under symmetric names, and inject both into all analyst prompts. Scope is limited to `screener/macro.py`, `analyst/claude_analyst.py`, `main.py` fallback dicts, and the corresponding tests.

</domain>

<decisions>
## Implementation Decisions

### Key Names
- **D-01:** Rename existing `spy_trend` dict key → `spy_trend_1m` everywhere (macro.py return value, main.py fallback dicts, all `macro_context.get("spy_trend")` calls in claude_analyst.py, test fixtures).
- **D-02:** New 1-year signal dict key is `spy_trend_1y`. Both keys follow the `spy_trend_{period}` pattern.
- **D-03:** Rename `format_spy_trend` → `format_spy_trend_1m` (or keep as a shared `format_spy_trend` helper accepting any return float — either is acceptable, planner decides). Both 1m and 1y use the same formatter logic.

### Calculation Method
- **D-04:** 1-year return uses the same endpoint-to-endpoint formula as 1-month: `(last_close - first_close) / first_close` using `spy.history(period="1y")`. No new calculation logic — reuse `format_spy_trend` formatter directly.
- **D-05:** Price-vs-MA deviation approach (from the design doc) is explicitly deferred to a future phase — note in deferred section.

### Prompt Display
- **D-06:** In all three prompt builders (`build_prompt`, `build_etf_prompt`, `build_sell_prompt`), the two signals render as:
  ```
  - SPY trend (1m): Bullish (+3.2%)
  - SPY trend (1y): Bearish (-8.5%)
  ```
  Label format: `SPY trend ({period}):` where period is `1m` or `1y`.

### Fallback Dicts
- **D-07:** Both `main.py` fallback dicts must be updated from `{"spy_trend": None, "vix_level": None}` to `{"spy_trend_1m": None, "spy_trend_1y": None, "vix_level": None}`.

### Claude's Discretion
- Whether `format_spy_trend` is renamed or kept as a shared helper — planner picks the cleaner approach.
- Test strategy: whether to add new tests for `spy_trend_1y` or extend existing parametrized tests.

</decisions>
