## What Was Built

`fetch_macro_context()` now returns a 3-key dict:
- `spy_trend_1m`: 1-month SPY return formatted as "Bullish (+3.2%)" or "Bearish (-1.4%)"
- `spy_trend_1y`: 1-year SPY return formatted with the same `format_spy_trend` helper
- `vix_level`: unchanged

All 3 prompt builders (`build_prompt`, `build_etf_prompt`, `build_sell_prompt`) now render two market context lines when both keys are present:
```
- SPY trend (1m): Bullish (+3.2%)
- SPY trend (1y): Bearish (-8.5%)
```

Both `except` fallback dicts in `main.py` use the new 3-key shape. All 8 test files updated to match.

## Decisions Made

- `format_spy_trend` kept as a shared helper called for both periods — no rename needed; it works for any fractional return float (per plan D-03 discretion).
- 1-year history fetched using the existing `spy` Ticker object (`spy.history(period="1y")`) — no second `yf.Ticker("SPY")` instantiation.
- `side_effect=lambda period: ...` pattern used in all updated test mocks to support multi-period history calls on the same mock object.
