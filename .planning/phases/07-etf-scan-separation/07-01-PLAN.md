<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| etf_watchlist.txt -> partition_watchlist | User-editable file read into ticker list |
| yfinance quoteType -> routing decision | External API response determines stock vs ETF classification |
| SQLite migration | Additive ALTER TABLE on production DB |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | etf_watchlist.txt | mitigate | `get_watchlist()` already strips whitespace and skips comments/blanks; ticker symbols are uppercased; no path traversal possible since path is hardcoded in main.py |
| T-07-02 | Spoofing | yfinance quoteType | accept | External data used only for classification routing, not for financial decisions; allowlist fallback provides a second check; misclassification is low-impact (ETF goes through stock filter or vice versa) |
| T-07-03 | Denial of Service | partition_watchlist | mitigate | yfinance rate limiting could block all lookups; allowlist fallback ensures classification still works when API is unavailable |
| T-07-04 | Information Disclosure | asset_type column | accept | Column value is 'stock' or 'etf' — no sensitive data; local SQLite only |
| T-07-05 | Tampering | SQLite migration | mitigate | Additive-only ALTER TABLE (never DROP/DELETE); try/except handles idempotent re-runs; DEFAULT 'stock' preserves existing rows |
</threat_model>
