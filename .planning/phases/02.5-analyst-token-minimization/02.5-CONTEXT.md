# Phase 2.5: Analyst Token Minimization — Context

**Gathered:** 2026-04-02
**Status:** Ready for planning
**Source:** PRD Express Path (docs/superpowers/specs/2026-04-01-analyst-token-minimization-design.md)

<domain>
## Phase Boundary

Prevent Gemini free-tier rate limit exhaustion (5 RPM / 20 RPD / 250k TPM) by:
1. Narrowing the analyst universe from ~500 S&P tickers to top 10 by EPS+ROE rank
2. Caching analyst results in SQLite keyed by (ticker, headline_hash) — skip API call on same news
3. Enforcing ≤5 RPM with a 12-second inter-call sleep in analyze_ticker
4. Replacing tenacity's fixed exponential backoff with a `_wait_for_retry` callable that parses Gemini's `retryDelay` hint from 429 bodies

This phase does NOT change the prompt format, response schema, or LLM provider. It does NOT batch tickers into a single call.

</domain>

<decisions>
## Implementation Decisions

### Universe Narrowing

- Add `get_top_sp500_by_fundamentals(n: int = 10) -> list[str]` to `screener/universe.py`
- Fetch `trailingEps` and `returnOnEquity` from yfinance for each S&P 500 ticker
- Score by combined rank: `eps_rank + roe_rank` (lower = better); return top N
- Cache ranked list in-memory (module-level variable) with 24h TTL — avoids repeated mass yfinance calls
- `run_scan()` in `main.py` replaces `get_sp500_tickers()` with `get_top_sp500_by_fundamentals(config.top_sp500_count)`

### Config

- Add to `Config` dataclass in `config.py`:
  - `top_sp500_count: int = int(os.getenv("TOP_SP500_COUNT", "10"))`
  - `analyst_call_delay_s: float = float(os.getenv("ANALYST_CALL_DELAY_S", "12.0"))`

### Analyst Cache — Schema

New table in `database/models.py`:
```sql
CREATE TABLE IF NOT EXISTS analyst_cache (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    ticker      TEXT NOT NULL,
    headline_hash TEXT NOT NULL,
    signal      TEXT NOT NULL,
    reasoning   TEXT NOT NULL,
    created_at  TEXT NOT NULL DEFAULT (datetime('now')),
    UNIQUE(ticker, headline_hash)
);
```

### Analyst Cache — Queries

Two new functions in `database/queries.py`:
- `get_cached_analysis(db_path: str, ticker: str, headline_hash: str) -> dict | None`
  — returns `{"signal": ..., "reasoning": ...}` on hit, `None` on miss
- `set_cached_analysis(db_path: str, ticker: str, headline_hash: str, signal: str, reasoning: str) -> None`
  — upserts using `INSERT OR REPLACE`

### Analyst Cache — Cache Key

`headline_hash` = `hashlib.sha256("\n".join(sorted(headlines)).encode()).hexdigest()`

Computed in `run_scan()` after `fetch_news_headlines()`. Cache invalidation is content-driven: changing headlines = new hash = fresh API call. No TTL needed.

### Cache Flow in run_scan()

```
headlines = fetch_news_headlines(ticker)
headline_hash = sha256_of(headlines)

cached = get_cached_analysis(config.db_path, ticker, headline_hash)
if cached:
    analysis = cached          # skip API call
else:
    analysis = analyze_ticker(ticker, info, headlines, config, client=client)
    set_cached_analysis(config.db_path, ticker, headline_hash, analysis["signal"], analysis["reasoning"])
```

### Inter-call Delay

`analyze_ticker()` in `analyst/claude_analyst.py` calls `time.sleep(config.analyst_call_delay_s)` **before** invoking `_call_api()`. This must be called even when config is passed from tests — tests that mock `_call_api` will still sleep unless `analyst_call_delay_s=0` is set.

At 12s/call, 20 tickers = ~4 minutes total scan time, at or under 5 RPM.

### Retry Delay Parsing

Replace tenacity's `wait_exponential(min=2, max=30)` in `claude_analyst.py` with:

```python
def _wait_for_retry(retry_state) -> float:
    exc = retry_state.outcome.exception()
    if exc and hasattr(exc, 'response'):
        try:
            delay_str = exc.response.json()['error']['details'][-1]['retryDelay']
            return float(delay_str.rstrip('s')) + 1
        except Exception:
            pass
    return min(2 ** retry_state.attempt_number, 60)
```

This targets Gemini's error body structure. For Anthropic/OpenAI, the fallback exponential path is used.

<specifics>
## Specific Ideas

- The in-memory 24h TTL cache for `get_top_sp500_by_fundamentals` should use a module-level tuple `(_cache_data, _cache_time)` pattern — same approach as any time-bounded memoization
- `INSERT OR REPLACE INTO analyst_cache` handles the upsert for `set_cached_analysis`
- The `_wait_for_retry` function accesses `exc.response.json()['error']['details'][-1]['retryDelay']` — wrap the entire parse in a broad `except Exception` since the structure varies across Gemini API versions

</specifics>
