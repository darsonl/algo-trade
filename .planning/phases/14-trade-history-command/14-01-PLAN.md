```python
def get_closed_trades(db_path: str, limit: int = 20) -> list[dict]:
    """Return up to `limit` closed sell-trades newest-first.

    Filter per D-05/D-06: WHERE side = 'sell' AND cost_basis IS NOT NULL — no JOIN.
    Pre-migration rows with NULL cost_basis are excluded consistent with get_trade_stats.

    Returns a list of plain dicts with keys: ticker, price (exit price), cost_basis (entry price),
    executed_at (ISO timestamp string). Empty list when no qualifying rows exist.
    """
    conn = get_connection(db_path)
    rows = conn.execute(
        """SELECT ticker, price, cost_basis, executed_at FROM trades
           WHERE side = 'sell' AND cost_basis IS NOT NULL
           ORDER BY executed_at DESC
           LIMIT ?""",
        (limit,),
    ).fetchall()
    conn.close()
    return [
        {
            "ticker": row["ticker"],
            "price": row["price"],
            "cost_basis": row["cost_basis"],
            "executed_at": row["executed_at"],
        }
        for row in rows
    ]
```

```python
def build_history_embed(trades: list[dict]) -> discord.Embed:
    """Build 'Trade History' embed — monospaced code-block table of last N closed trades.

    Caller MUST pass a non-empty list; empty-state is handled at the command layer.
    Columns per D-02: Ticker  Entry    Exit     P&L%    Date (ISO YYYY-MM-DD per D-03).
    P&L% prefixed with '+' for gains, '-' for losses (embedded sign format).
    """
    header = "Ticker  Entry     Exit      P&L%     Date"
    lines = [header]
    for t in trades:
        ticker = t["ticker"]
        entry = t["cost_basis"]
        exit_price = t["price"]
        pnl_pct = (exit_price - entry) / entry * 100.0
        sign = "+" if pnl_pct >= 0 else "-"
        pnl_str = f"{sign}{abs(pnl_pct):.1f}%"
        # ISO date only — strip time portion if executed_at is 'YYYY-MM-DDTHH:MM:SS'
        date_iso = str(t["executed_at"])[:10]
        lines.append(
            f"{ticker:<7} {entry:>8.2f}  {exit_price:>8.2f}  {pnl_str:>8}  {date_iso}"
        )
    table = "\n".join(lines)
    embed = discord.Embed(title="Trade History", color=discord.Color.blurple())
    embed.description = f"```\n{table}\n```"
    return embed
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discord user -> Discord API -> bot | Command invocation crosses from untrusted user into the bot process; payload is only the slash command name (no arguments this phase). |
| bot process -> SQLite | DB read — `/history` issues a parameterised SELECT; no user-supplied values reach the query. |
| bot process -> Discord API | Outbound embed render; content is server-controlled (DB rows formatted server-side). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Spoofing | Discord user invoking `/history` | accept | Discord app_commands enforces user identity via Discord OAuth2; phase adds no new auth surface. No secret data exposed (public trade history already visible via `/stats` and `/positions`). |
| T-14-02 | Tampering | SQL query in `get_closed_trades` | mitigate | Use `?` parameterised binding for `limit` (Task 1 action explicitly forbids f-string interpolation). No user-supplied values enter the query — `limit` is a hardcoded internal default of 20. |
| T-14-03 | Repudiation | Read-only command | accept | `/history` is non-mutating (SELECT only). No audit trail needed beyond existing Discord message log. |
| T-14-04 | Information Disclosure | Embed content exposes P&L data in channel | accept | Same channel already displays `/positions` (P&L%) and `/stats` (win rate). No new data class is exposed. Operator is sole user of the private bot channel per PROJECT.md. |
| T-14-05 | Denial of Service | Large result set or slow DB blocks gateway heartbeat | mitigate | `LIMIT 20` caps result set; `asyncio.to_thread` wraps the DB call (Task 3 action + acceptance grep) to keep the Discord gateway heartbeat unblocked — consistent with project-wide asyncio.to_thread rule in CLAUDE.md. |
| T-14-06 | Elevation of Privilege | Discord button/command grants no new privilege | accept | `/history` is read-only; no order placement, no DB writes, no config change. Disposition: no attack surface. |
| T-14-07 | Information Disclosure | Embed description exceeds Discord 4096-char limit and fails to render | mitigate | Row format is bounded (~50 chars/row × 20 rows ≈ 1000 chars + fence overhead << 4096). D-01 explicitly verified headroom. No mitigation code required — bounded by `LIMIT 20`. |
</threat_model>
